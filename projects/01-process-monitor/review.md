## 🐛 Критические ошибки

### 1. Опечатка в имени переменной — `procesMemory` вместо `processMemory`

```go
// Написано:
processIndex := 0
procesMemory := 0  // опечатка!
```

Именование — это не эстетика, это корректность. В большом проекте такая опечатка превращается в баг, который компилятор не поймает.

### 2. Тип памяти `[]int` вместо `[]float64`

```go
// Написано:
mbytes := []int{100, 200, 150, 300, ...}

// Правильно:
mbytes := []float64{100.5, 200.3, 150.0, 300.7, ...}
```

ТЗ явно требовало `float64`. В реальности память измеряется дробными значениями. Потеря точности.

## 🔧 Улучшения

### 1. Длинная цепочка `if/else` вместо `switch`

```go
// Написано:
if name == "Running" {
    runningProcesses++
} else if name == "Sleeping" {
    ...
}

// Правильно:
switch name {
case "Running":
    runningProcesses++
case "Sleeping":
    sleepingProcesses++
case "Stopped":
    stoppedProcesses++
case "Zombie":
    zombieProcesses++
    pidsZombie = append(pidsZombie, pids[i])
}
```

### 2. Нет констант для строк статусов

```go
const (
    StatusRunning  = "Running"
    StatusSleeping = "Sleeping"
    StatusStopped  = "Stopped"
    StatusZombie   = "Zombie"
)
```

Магические строки разбросаны по коду — опечатка в одном месте создаст баг, который компилятор не поймает.

### 3. Вывод не отформатирован

Использовать `%-20s`, `%6d`, `%.1f` для выравнивания колонок.
